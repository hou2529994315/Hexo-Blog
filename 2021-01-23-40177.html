基础查询

格式：SELECT 查询列表 FROM 表名；
查询列表可以是：表中的字段、常量值、表达式、函数
查询的结果是一个虚拟的表格

建议在查询开始的地方选中目标数据库USE myemployees;

  1. 查询表中的单个字段

    SELECT last_name FROM employees;

  2. 查询表中的多个字段

    SELECT last_name,salary,email FROM employees;

  3. 查询表中的所有字段

    SELECT * FROM employees;

  4. 如果字段名和关键字重名，要用``括起来

    SELECT `NAME` FROM stuifo;

  5. 查询常量值

    SELECT 100;
SELECT "John";

  6. 查询表达式

    SELECT 100%98;

  7. 查询函数

    SELECT VERSION();

  8. 设置别名
    作用：便于理解、区分重名

    SELECT 100%98 AS 结果;
SELECT last_name AS 姓, first_name AS 名 FROM employees;

    简写

    SELECT last_name 姓,first_name 名 FROM employees;

    别名可以添加双引号,产生存在空格的别名

    SELECT salary AS "out put" FROM employees;

  9. 去重（DISTINCT）

    SELECT DISTINCT department_id FROM employees;

  10. +号的作用
    mysql中+号只有一个功能：运算符
    “100”+90;字符型加数值型，则先将字符型转换成数值型。若转换成功，则加法运算。若转换失败，则转换为0
    null + 数值型 = null
    字符串拼接需要用函数CONCAT().

    SELECT
CONCAT(last_name, first_name ) AS 姓名 
FROM
employees;

  11. 拼接employee的全部列，并用逗号连接

    SELECT
 IFNULL( commission_pct, 0 ) AS 奖金率,
 commission_pct 
FROM
 employees;
SELECT
CONCAT(
    first_name,
    ',',
    last_name,
    ',',
    job_id,
    ',',
IFNULL( commission_pct, 0 )) AS out_put 
FROM
employees;

条件查询

语法：

SELECT
        查询列表
FROM
        表名
WHERE
        筛选条件;逐行筛选

执行顺序：
1. from 先查看当前数据库有没有这张表
2. where 筛选，筛选出满足要求的行
3. select 查询需要的类

分类：

  • 按条件表达式筛选
    条件运算符 ：> < = <> >= <=
  • 按逻辑表达式筛选
    逻辑运算符：
    • && || !
    • and or not
  • 模糊查询
  • like
  • between and
  • in
  • is null

  1. 工资大于12000的员工信息

    SELECT
 * 
FROM
 employees 
WHERE
 salary > 12000;

  2. 部门标号不等于90号的员工名和部门编号

    SELECT
 last_name,
 department_id 
FROM 
 employees 
WHERE
 department_id != 90;
 department_id <> 90;

  3. 查询工资在10000到20000之间的员工名、工资和奖金

    SELECT
 last_name,
 salary,
 commission_pct 
FROM
 employees 
WHERE
 salary >= 10000 
 AND salary <= 20000;

  4. 查询员工名中包含字符a的员工信息(默认不区分大小写)
    like一般和通配符搭配使用
    通配符有：

  • %，任意多个字符，包含0个字符
  • —，任意单个字符
    SELECT
  *
FROM
  employees
WHERE
  last_name LIKE '%a%';

  1. 查询员工名第三个字符为n,第五个字符为l的员工名

    SELECT
 last_name
FROM
 employees
WHERE
 last_name LIKE '__n_l%';

  2. 6.1. 查询第二个字符为_的员工名(转义运算符 )

    SELECT
last_name
FROM
employees
WHERE
last_name LIKE '_\_%';

    6.2. 也可以随意指定转义符

    SELECT
last_name
FROM
employees
WHERE
last_name LIKE '_$_%' ESCAPE '$';

  3. 查询员工编号在100到120之间的员工信息
    between and可以提高语句的简洁度，它包含两侧数指的临界值

    SELECT
 *
FROM
 employees
WHERE
 department_id BETWEEN 100 AND 110;

  4. 查询员工的工种编号是 ‘FI_MGR’, ‘FI_ACCOUNT’, ‘AC_MGR’, ‘AD_VP’的员工信息
    in:判断某字段的值是否属于in列表中的某一项
    特点

    • 提高语句简介度
    • in列表中的值类型必须一致或兼容
    • in列表中不支持通配符
      SELECT
last_name,
job_id
FROM
employees
WHERE
job_id IN('FI_MGR', 'FI_ACCOUNT', 'AC_MGR', 'AD_VP');

  5. 9.1. 查询没有奖金的员工名和奖金率
    is null: < > = 不能用于判断null值

    SELECT
 last_name,
 commission_pct
FROM
 employees
WHERE
 commission_pct IS NULL;

    9.2. 查询有奖金的员工名和奖金率

    SELECT
 last_name,
 commission_pct
FROM
 employees
WHERE
 commission_pct IS NOT NULL;

  6. 安全等于 <=>

    SELECT
last_name,
commission_pct
FROM
employees
WHERE
commission_pct <=> NULL;

对比
IS NULL :仅仅可以判断null值，可读性较高
<=> :既可以判断NULL值，也可以判断普通数值，但是可读性较低。建议用IS NULL

排序查询

语法：

SELECT 查询列表
FROM   表
[WHERE 筛选条件]
ORDER BY 排序列表 [ASC、DESC];

特点：

  • 默认升序
  • ORDER BY 子句支持单个字段、多个字段、表达式、函数、别名的排序
  • ORDER BY 子句一般是放在查询语句的最后边。LIMIT子句除外。

1.
1.1 查询员工信息，要求工资从高到底排序

SELECT * FROM employees ORDER BY salary DESC;
1.2 从低到高排序,ASC可省略，即默认排序是升序
SELECT * FROM employees ORDER BY salary ASC;
SELECT * FROM employees ORDER BY salary;

  1. 查询部门编号>=90的员工信息，按入职时间排序

    SELECT * 
FROM employees
WHERE department_id >= 90
ORDER BY hiredate ASC;

  2. 按年薪的高低排序显示【按表达式排序】

    SELECT *, salary*12*(1+IFNULL(commission_pct,0)) AS 年薪
FROM employees
ORDER BY salary*12*(1+IFNULL(commission_pct,0));

  3. 支持按别名排序

    SELECT *, salary*12*(1+IFNULL(commission_pct,0)) AS 年薪
FROM employees
ORDER BY 年薪;

  4. 按姓名的长度显示员工的姓名和工资【按函数排序】

    SELECT LENGTH(last_name) 字节长度, last_name, salary
FROM employees
ORDER BY 字节长度;

  5. 先按工资排序，再按员工编号排序【按多个字段排序】

    SELECT *
FROM employees
ORDER BY salary ASC, employee_id DESC;
