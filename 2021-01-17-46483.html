ElasticSearch(1) Win10+Docker下安装ElasticSearch

Docker ElasticSearch ik分词器
ElasticSearch
发布日期:   2021-01-17
文章字数:   385

在Docker中安装ElasticSearch

docker pull docker.elastic.co/elasticsearch/elasticsearch:7.10.0 # 拉取es镜像文件
docker run -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" -e "ES_JAVA_OPTS=-Xms256m -Xmx512m" --name es -d docker.elastic.co/elasticsearch/elasticsearch:7.10.0 # 启动单个节点集群，限制容器大小，命名为es，后台运行

执行成功后可以用docker ps可以看到容器正在运行中

并且可以打开链接进行测试http://localhost:9200/
如果出现下图的情况，则说明ElasticSearch的容器已经运行成功了。

在ElasticSearch中添加ik分词器

Elasticsearch中默认提供了一个标准分词器，但是该分词器对中文并不友好，该分词处理中文时是一元分词器。因此需要在ElasticSearch中添加新的分词器插件（本文以ik分词器为例）。

  1. 下载和ElasticSearch配套的ik插件https://github.com/medcl/elasticsearch-analysis-ik/releases。并将解压后将文件夹重名为ik。

  2. 进入容器

    docker exec -it 8cf1c /bin/bash # 其中8cf1c为容器id

    运行结果：

  3. 进入插件目录，可以看到插件目录是空的

    cd ..
cd ..
cd /usr/share/elasticsearch/plugins/  # 返回根目录后进入插件目录

  4. 将插件文件夹复制到容器内,并重启容器

    docker cp e:\elasticsearch\ik 8cf1c:\usr\share\elasticsearch\plugins # 第一个路径是本机路径，第二个路径是容器的文件路径，8cf1c是运行中的es容器
docker restart 8cf1c

    用Postman测试分词效果，可以看到标准分词的结果是一元分词，ik分词的结果不是一元分词。
